## Prioridades criticas para alcanzar 5.0

1. Capturar SonarCloud actual con Quality Gate, coverage >80%, duplicidad, mantenibilidad, reliability y seguridad.
2. Ejecutar y capturar el pipeline Docker CI/CD con publicacion real en DockerHub y despliegue Compose desde imagenes versionadas.
3. Exportar datos reales de GitHub con `gh`, cerrar o justificar milestones/issues pendientes y actualizar las metricas Scrum generadas.
4. Probar Twilio con credenciales privadas y capturar SMS recibido, dashboard Twilio y log backend sin secretos.
5. Rotar cualquier credencial que haya sido versionada y dejar `.env` fuera de futuros commits; la limpieza de historial debe decidirla el equipo.
6. Crear tag/release final o documentar explicitamente por que no existe, porque la rubrica pide control de versiones organizado.
